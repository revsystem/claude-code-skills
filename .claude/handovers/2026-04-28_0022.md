## 決定事項

- skills 管理は `gh skill install revsystem/claude-code-skills --agent claude-code --scope user` に移行する。npx skills は廃止。
- `install.sh` は hooks/agents の管理専用に整理する。`TYPE:NAME` 引数は `agents:` と `hooks:` のみ対応。
- `gh skill` はファイルをコピーするため開発時の即時反映ができない。開発中は `--from-local` フラグで都度再インストールする旨を README に注記した。

## 捨てた選択肢と理由

- install.sh にスキル処理を残しつつ npx を gh skill に差し替える案: スクリプトの肥大化を避けるため却下 — gh skill に一本化して install.sh を hooks/agents 専用に整理する方針を採用
- npx skills との並存案: ユーザーから「npx コマンドの案内は廃止」と明示されたため却下

## 学び

- `gh skill publish --dry-run` で既存スキルのバリデーションができる。現行スキルは `license` フィールド欠如の warning 2件のみでパスする。
- `gh skill` は Agent Skills 標準仕様に準拠しており、現行の `skills/*/SKILL.md` 規約がそのまま互換。
- `skills/handover/SKILL.md` の `hooks:` フィールドはプロジェクト独自仕様（install.sh の `resolve_hooks()` が解析）であり、Agent Skills 標準の `hooks:` フィールド（PreToolUse/PostToolUse 型）とは別物。`gh skill` はこのフィールドを無視する（エラーにならない）。
