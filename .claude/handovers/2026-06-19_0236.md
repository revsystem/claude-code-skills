## 決定事項
- 取り込み範囲は AI-DLC の Inception 側に限定する。採用は #1 深度レベル / #2 着手前ワークフロー宣言 / #3 選択式残決定事項 / #4 ユニット・グルーピング（論点1=B）
- 選択式フォーマットはテンプレート内に直書きする。参照ファイルは新設しない（論点2=A、シンプルさ優先）
- 深度宣言は standard/comprehensive のみ。minimal は省略の一言（論点3=B、小修正の体験を軽く保つ）
- リリース採番は semver。機能追加（feat）なのでマイナーバンプ v1.3.0
- リリースタグはリモート main 同期後に作成し、タグSHA=リモート main HEAD を確認する（CLAUDE.md の gh skill 規約）

## 捨てた選択肢と理由
- AI-DLC の全面移植: Construction/Operations（コード生成・ビルド/テスト・運用）が本スキルの「don't implement yet」境界と衝突し別物の大型ツールになる — Inception 側のみ選択的に取り込み
- #5 NFR レンズ: spec/plan テンプレートに性能・セキュリティ観点が既にあり実質重複・費用対効果が低い — 見送り（論点1=B で対象外）
- #6 Extensions（security/testing/resiliency の opt-in）: 本スキルを重くする — 見送り
- audit.md / aidlc-state.md / session-continuity 機構: この repo の handover スキルと重複。AI-DLC 自身の「重複排除」テネットにも反する — 取り込まない
- 常時適用の steering-rules 起動モデル（モデル自律起動）: 本スキルは意図的に `disable-model-invocation: true` — 起動方式は変更しない
- 成果物配置 `aidlc-docs/`: repo 規約の `.claude/docs/...` を維持 — 却下
- 参照ファイル `references/decision-format.md` 新設（論点2 B案）: ファイルとロード手順が増える — テンプレ直書き（A案）に
- 全深度で深度宣言を必須化（論点3 A案）: 小修正が冗長になる — standard/comprehensive のみ宣言（B案）に

## 学び
- AI-DLC = AWS Labs `awslabs/aidlc-workflows`。AIコーディングエージェント向け steering rules 集で、Inception→Construction→Operations の3フェーズ適応型。深度（minimal/standard/comprehensive）・質問駆動（A/B/C/D/E + `[Answer]:`）・human-in-the-loop・extensions opt-in が特徴。research-plan-annotate と設計思想（コード前計画・承認ゲート・ファイル成果物）を共有する

- セッション開始時に注入される slash command の本文（旧キャッシュ）と、ディスク上のインストール版・リポジトリ正本が三者三様にずれることがある。改修対象は常にリポジトリ正本を基準にする
