## 決定事項

- Skills ではなく Agents として配置する: Skills の frontmatter は `mcpServers` 未サポート、Agents は frontmatter インライン定義でき自己完結できるため
- 自律 spawn 方式（スラッシュコマンドではなく Claude が description を判断して spawn）を採用した
- MCP 設定は Agent frontmatter の `mcpServers` マッピング形式でインライン定義。settings.json への追記は不要
- `tools` に `Bash` を含めない。コードレビューのみを行い terraform コマンドの実行は行わないという意図的な設計
- `claude-code-plugins` 側の terraform-code-reviewer はそのまま残す（両リポジトリに共存）
- 本文から「使用例」「MCPサーバー連携」セクションを削除。コードブロック外の太字を除去し、コードブロック内の絵文字・太字は保持

## 捨てた選択肢と理由

- スラッシュコマンド（Skills）方式: Skills の frontmatter は `mcpServers` 未サポートのため MCP 依存の配布に不向き — Agents + 自律 spawn を採用
- install.sh が settings.json に MCP を自動注入する方式: ユーザーの settings.json を破壊するリスクがある — Agents frontmatter インライン定義で完全自己完結する方式を採用
- mcps/ ディレクトリ + symlink でユーザーが settings.json に1行追記する方式（アプローチC）: Agents frontmatter インライン定義でそもそも不要になったため廃案
- `~/.claude/agents/.mcp.json` でエージェント専用 MCP 設定する案: Claude Code 非サポートであることを調査確認 — frontmatter mcpServers インライン定義を採用

## ハマりどころ

- 設計書の `mcpServers` をリスト形式で記述していた → マッピング形式が正しい仕様（`aws-terraform-mcp-server:` 直下に `type:` 等）→ spec review で検出して修正
- Agent 実装時にコードブロック内の太字（`❌ **重大な問題**:` 等）を誤って除去した → spec review で検出 → `fix:` コミットで復元

## 学び

- Claude Code の Agents frontmatter は `mcpServers` をマッピング形式でインライン定義できる。定義したサーバーは Agent spawn 中のみ接続され終了時に切断される
- Skills の frontmatter は `mcpServers` 未サポート。Skills が MCP を使うにはセッションレベルでグローバル設定済みが必要
- Claude Code の `~/.claude/agents/` に置かれた Agent は settings.json 変更なしに Claude が自律的に spawn 可能。description の文言が spawn 判断の鍵になる
- `mcpServers` のリスト形式（`- key:` 形式）は Claude Code agents frontmatter では非サポート。マッピング形式（`key:` 直下に属性）が正しい
