## 決定事項

- テンプレートへのセクション追加は1つのみ（`リポジトリ状態`）。シンプルさ維持のため独立セクションを増やしすぎない
- セッション背景は独立セクションにせず「今回やったこと」の1行目に吸収する
- CLAUDE.mdのワークフロールールは「宣言的」より「手順的」に書く方が見落とし防止に効果的

## 捨てた選択肢と理由

- セッション背景を独立セクション化する案: セクションが増えすぎて記述負担が高まる — 「今回やったこと」の1行目に吸収する方式を採用
- research-plan-annotateスキルに「ブランチ作成」ステップを追加する案: スキル利用者が必ずしもこのルールを必要とするわけではない — CLAUDE.mdのルール記述を手順的に改訂する方針を採用

## ハマりどころ

- main ブランチで直接 SKILL.md を編集してしまった → CLAUDE.md のワークフロールールが「作業を行う際は」という宣言的記述で、編集開始前に頭に浮かびにくかった → ブランチを後から作成して移し、CLAUDE.md のルールを「ファイルを編集する前に git branch で確認する」という手順的記述に改訂
- Skill ツールで `/handover` を実行すると同一セッション中の SKILL.md 変更が反映されない → Claude Code はセッション開始時にスキルファイルをキャッシュする仕様 → 新セッションを起動してテストすることで解決

## 学び

- Claude Code のスキルファイルはセッション開始時にキャッシュされる。同一セッション中にスキルファイルを編集しても Skill ツールには反映されない。スキル変更のテストは新セッションで行う必要がある
- CLAUDE.md のルールは「〜すべき」という宣言より「〜する前に〜を確認する」という手順として書く方が、別のスキルのフローに乗っているときでも見落としにくい
