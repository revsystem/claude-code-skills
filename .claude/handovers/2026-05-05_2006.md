## 学び

- `gh skill install` は `~/.claude/skills/` にシンボリックリンクを作るが、旧 install.sh の `install_skill()` も同じく `~/.claude/skills/` にシンボリックリンクを作るため、移行前後でリンク先が同じなら外見上の区別がつかない
- `~/.claude/.claude/` という入れ子ディレクトリが存在する。これは `~/.claude/` をプロジェクトルートとして Claude Code を動かしたセッションが作成した `settings.local.json` の置き場であり、設計上は想定外の副産物
