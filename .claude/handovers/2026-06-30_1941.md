## ハマりどころ
- `chezmoi apply` がデプロイ先の out-of-band 変更を検知し上書き確認の TTY を開けず失敗 → statusline-setup エージェントが live を直接編集していたため → `chezmoi apply --force` で解決
- `mv`/`cp`/`rm` がシェルで `-i` エイリアスに化け、上書き確認で停止しスクリプトが進まない → 対話エイリアス → `/bin/cp -f` / `command mv -f` / `command rm -f` で回避
- 一時ファイルを chezmoi ソースディレクトリ内に書いたら `executable_statusline.sh.tmp` を chezmoi が管理対象 `.claude/statusline.sh.tmp` として拾った → ソース dir 内の任意ファイルは管理エントリ扱い → 一時ファイルはソース外（scratchpad）に書き、削除して解決
- awk 置換 + mv でファイルの実行ビットが 755→644 に落ちた → コミット差分にモード変更ノイズ → `chmod 755` で復元してからコミット
- フォルダグリフが表示されないように見えた → 素の Cascadia Code は Powerline 記号（U+E0A0）は持つが Font Awesome 系（U+F114/F115）は持たない → Nerd Font（Cascadia Code NF 等）が必要。ユーザー環境では表示確認できた（Nerd Font 導入済み）

## 学び
- bash の `$''` は C.UTF-8 ロケールでランタイムに UTF-8 バイト（EF 84 95）へ展開され、リテラルグリフと同一表示になる
- chezmoi の `executable_` プレフィックスはターゲットの実行ビットを設定する。ソースファイル自体のモード変更は chezmoi 動作には無関係だが git 差分には出る
- `chezmoi status` の `MM` は「ターゲットが前回 apply 後に変更」かつ「apply で変わる（ソースと差異）」の両方を意味する
- chezmoi ソース dir 内に書いた任意ファイルは管理エントリとして解釈されるため、一時ファイルをそこに置かない
- statusline コマンドは描画ごとに再実行される（純粋なセッション開始時キャッシュではない）一方、スキル本文はセッション開始時にキャッシュされる
- スキルのゲートはハーネスのパーミッション層ではなく SKILL.md の指示で担保される会話レベルの停止点。auto mode はパーミッション層を切るだけで機械的にゲートを無効化しないが、意図の衝突・安全網喪失・無人実行で human-in-the-loop は実質的に崩れる
