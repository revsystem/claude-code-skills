# handover スキル 外部引き継ぎ設計の知見取り込み 実装計画

日時: 2026-05-29
関連調査: ../research/2026-05-29-handover-coldread-adoption-research.md

## 概要

別プロジェクトの引き継ぎ設計（参考実装）から3点（Cold-read order / Status・Drives / Memory references）を handover に取り込む。トラック機構は非適用。各点は軽い1〜数行に収め、handover の per-session・自動起動・小サイズという性質を壊さない。反映先は記入例ブロック（統合済みの雛形）と、必要なら Process / Quality Rules。

## アプローチ

新セクション乱立を避け、可能な限り既存8セクションへ統合する（handover を小さく保つ方針＝assets 非分離の判断と整合）。記入例は実話で充填し、各見出し直下に `<!-- -->` ルールを置く既存形式を踏襲する。参考実装は外部由来のため固有値は持ち込まず、汎用パターンとして反映する。

## 変更内容

### 変更1: Status / Drives を冒頭に追加

対象: `skills/handover/SKILL.md` 記入例ブロックの日時直後

セッションの立ち位置と駆動 Issue/PR を冒頭で一望できるようにする。日時の下に2行追加する案。

```markdown
日時: 2026-03-17 17:26
状態: Working（main未マージ / レビュー前 など）
駆動: Issue #1「セッション引き継ぎ自動化」, PR #2
```

`<!-- -->` ルール例: 「状態は Working/レビュー前/ブロック中/完了 等。駆動はこのセッションが対応する Issue/PR を1行（無ければ「なし」）」。

### 変更2: Cold-read order（最初に読む順）

対象: `skills/handover/SKILL.md`（配置は残決定事項#1）

予備知識ゼロの次セッションが最速で文脈復元できる読書動線を、所要時間目安付きで並べる。3〜5項目。

```markdown
## 最初に読む（cold-read順）
<!-- 予備知識ゼロで再開する人がこの順に読めば文脈を復元できるファイルを3〜5件。所要時間の目安を添える -->
1. `.claude/docs/plans/2026-...-plan.md` — 実装計画とタスク状況（≈3分）
2. `.claude/docs/research/2026-...-research.md` — 背景と調査根拠（≈5分）
3. `src/foo.ts` — 今回の変更の中心（≈3分）
```

既存「関連ファイル」（パス＋一言の全体リスト）との重複は、cold-read を「読む順の最小セット」、関連ファイルを「網羅リスト」と役割分担する（残決定事項#1で配置と統合方針を確定）。

### 変更3: Memory references（関連メモリ）

対象: `skills/handover/SKILL.md`（配置は残決定事項#2）

次セッションが関連知識へ最短到達できるよう、`~/.claude/projects/.../memory/` の関連エントリを指す。

```markdown
## 関連メモリ
<!-- このセッションに関連する ~/.claude/.../memory/ のエントリ名。無ければ省略可 -->
- [[handover-skill-design]] — handover の設計判断
```

### 変更4: Process / Quality Rules の追従（最小）

対象: `skills/handover/SKILL.md` Process 手順4 / Quality Rules

記入時に新要素を拾うよう、Process 手順4の確認項目に「cold-read順の決定」「駆動 Issue/PR と関連メモリの確認」を1行ずつ追記する。Quality Rules には横断原則のみ残す方針を維持し、セクション個別ルールはコメント側に置く。

## 影響範囲

- 既存テスト: 本文は自動テスト非対象。影響なし。
- 行数: 96行 → 概算110〜120行（3点とコメントで +15〜20行）。assets 非分離方針は維持。
- 配布: `gh skill install` で本文コピーのみ。構成変化なし。
- frontmatter: 無変更。

## 考慮事項

- 低コンテキスト自動起動時の記入負荷。3点を必須化すると生成が重くなる懸念。各点は「該当なしなら省略可／1〜数行」に倒すか要判断（残決定事項#3）。
- cold-read order と 関連ファイル・次にやること[起点] の三者の重複整理。次にやること[起点]は「最初の操作」、cold-read は「最初に読むファイル」、関連ファイルは「網羅リスト」と役割を分ける。
- スキルはセッション開始時キャッシュ。挙動確認は新セッション。

## 決定事項（annotation で確定）

1. Cold-read order の配置 = 案A。新セクション「## 最初に読む（cold-read順）」を上部（リポジトリ状態の直後）に追加。既存「関連ファイル」は網羅リストとして残す。
2. Memory references の配置 = 案A。新セクション「## 関連メモリ」を末尾付近に任意（省略可）で追加。
3. 各点の必須/任意 = 案A。Status/Drives と Cold-read order は必須セクション。関連メモリは任意（該当なしなら省略可）。
