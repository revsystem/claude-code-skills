# handover SKILL.md テンプレート/記入例 重複解消 実装計画

日時: 2026-05-29
関連調査: ../research/2026-05-29-handover-template-duplication-research.md

## 概要

handover の SKILL.md にある「テンプレート（25-62行）」と「記入例（75-136行）」は同じ8セクションの骨格を二重に持つ。これを単一の「注釈付き完成例」に統合し、骨格の二重定義と書き方指示の三重分散（括弧内ヒント／Quality Rules／記入例）を解消する。few-shot 品質を保つため、満杯の実例は残し、各セクション直下に1行の書き方ルールをコメントで埋め込む。

## アプローチ

統合方式を採る（assets 分離は採らない）。理由は調査のとおり: 137行は小さく分離の progressive-disclosure 利得が薄い／handover は低コンテキスト時に inline 実行されるため Read 往復を増やしたくない／同リポジトリの research-plan-annotate も「構成にヒントをインライン併記」方式で別掲例を持たない。

採用しない案:

- assets/EXAMPLE.md への分離 — 配布ファイル増・Read 往復増・例の読み飛ばしリスク。前回セッションでも非推奨と結論済み。
- 記入例の単純削除 — few-shot のトーン・粒度ガイドを失い出力品質が落ちる。

## 変更内容

### 変更1: テンプレートと記入例を単一ブロックに統合

対象: `skills/handover/SKILL.md`（`## テンプレート` 23-62行 と `## 記入例` 73-136行）

`## テンプレート` と `## 記入例` の2見出しを廃し、`## 記入例（このまま雛形として使う）` の1見出しに統合する。中身は現在の記入例（handover 開発実話で充填済み）をベースに、各セクション見出しの直下へ1行の書き方ルールを HTML コメントで添える。これによりヒントと実例が同居し、骨格は1回しか定義されない。

```markdown
## 記入例（このまま雛形として使う）

\`\`\`markdown
```

### 変更2: Quality Rules の重複指示を整理

対象: `skills/handover/SKILL.md`（`## Quality Rules` 64-71行）

各セクション固有の before→after 例（体言止め禁止など）は変更1のコメントへ移動するため、Quality Rules には全セクション横断の原則のみ残す。残す候補: 事実ベースで書く／「捨てた選択肢」が最重要／該当なしは「なし」／簡潔に箇条書き中心。セクション個別の言い換え例は削除し、コメント側に集約する。

## 影響範囲

- 既存テスト: `tests/test_install.sh` は install.sh の symlink 回帰テストで SKILL.md 本文を検証しない。影響なし（要確認）。
- 配布: `gh skill install` は SKILL.md をコピーするのみ。ファイル構成は単一のまま変わらず、配布物への影響なし。
- 行数: 137行 → 概算85〜95行（記入例の二重骨格 約50行削減）。
- frontmatter（name/description）は変更しない。スキル発火条件に影響なし。

## 考慮事項

- HTML コメント `<!-- -->` は生成される実ノートにそのまま混入させない運用にする（コメントは「雛形の説明」であり、複製時に削除する想定）。この前提を統合ブロック冒頭の一文で明示する。
- few-shot の具体性（実話）を損なわないよう、記入例本文の固有名詞・具体ログは温存する。
- スキルファイルはセッション開始時にキャッシュされる（CLAUDE.md記載）。編集後の動作確認は新セッションで行う。

## 決定事項（annotation で確定）

1. ヒントの埋め込み形式 = 案A（HTML コメント `<!-- ルール -->` をセクション直下に置く）。混入緩和として、統合ブロック冒頭に「複製時はコメント行を削除する」と明示する（変更1に反映済み）。
2. Quality Rules の扱い = 案A（全セクション横断の原則のみ残し、セクション個別の言い換え例はコメントへ集約。変更2の方針どおり）。
3. 統合見出しの名称 = `## 記入例（このまま雛形として使う）`（変更1に反映済み）。
